# QuickBooks Online Sandbox Setup

Do not place credentials in chat, source code, GitHub, frontend variables, or documentation. Use the local backend environment for development and AWS Secrets Manager for deployed environments.

## 1. Create or open the Intuit app

1. Sign in at `developer.intuit.com` with the Savians-owned account.
2. Open **My Hub -> App dashboard**.
3. Open the Savians integration app, or create one with the **QuickBooks Online Accounting** scope.
4. Open **Keys and credentials**, choose **Development**, and enable **Show credentials**.
5. Store the Development Client ID and Client Secret securely.

Development credentials work only with sandbox companies. Production credentials are separate.

## 2. Select a sandbox company

1. Open **My Hub -> Sandboxes**.
2. Use the automatic sandbox or create a US QuickBooks Online Plus/Advanced sandbox.
3. Choose one shared staging accounting company.

Sandbox API base URL: `https://sandbox-quickbooks.api.intuit.com/v3`.

## 3. Get the realm ID and refresh token

1. Open Intuit's **OAuth 2.0 Playground**.
2. Select the development app/sandbox.
3. Select `com.intuit.quickbooks.accounting`.
4. Choose **Get authorization code**, connect the sandbox, and choose **Get tokens**.
5. Store the generated Realm ID and refresh token securely.

Access tokens last about one hour and are generated by the backend. Every refresh may return a new refresh token, so the backend must persist the newest one atomically.

## 4. Get the Tax Assessment item ID

1. Open the sandbox company.
2. Create or locate the Product/Service named **Tax Assessment**.
3. During the Phase 3 connection test, query the QuickBooks Item entity and record its numeric ID as `QB_SERVICE_ITEM_ID_TAX_ASSESSMENT`.
4. Confirm its income-account mapping and that test invoices total exactly USD 2,997.

The display name is not the API item ID.

## 5. Configure development webhooks

1. In the Intuit app, open **Webhooks -> Development**.
2. Add the staging HTTPS webhook endpoint.
3. Subscribe to required Invoice and Payment operations.
4. Save and securely store the Development verifier token.

Sandbox and production webhooks/verifier tokens are separate.

## Required backend variables

- `QB_ENVIRONMENT=sandbox`
- `QB_CLIENT_ID`
- `QB_CLIENT_SECRET`
- `QB_REFRESH_TOKEN`
- `QB_REALM_ID`
- `QB_WEBHOOK_VERIFIER_TOKEN`
- `QB_SERVICE_ITEM_ID_TAX_ASSESSMENT`
- `QB_MINOR_VERSION`
- `QB_BASE_URL=https://sandbox-quickbooks.api.intuit.com/v3`

## Ready check

Setup is complete when the backend can refresh a token, fetch CompanyInfo for the expected realm, query the service item, create and re-fetch one USD 2,997 test invoice, and validate a signed sandbox webhook.
